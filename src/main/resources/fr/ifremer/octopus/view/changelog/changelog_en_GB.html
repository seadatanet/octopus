Change log

Version 1.7.0 - 2021/09/20

Adds-on
  • Conversion of EGO1.2, 1.3 and 1.4 glider files to SeaDataNet netCDF (CFPoint) format.
  • Use of "BODV" and "MODV" formats in the coupling table.
Corrections
  • More information in the log when updating vocabularies.
  • Message in the log on external error (e.g. if a time-out occurs during the vocabulary update).
  • "Update lists" button changed to "Update vocabularies" in the "Settings" window.
  • Check that unit P06::TISO is used for P01::ENDX8601.
  • Conversion from MedAtlas to ODV: the station comments are kept.
  • Corrections de labels, messages, typos, ...

Version 1.6.0 - 2021/01/07

Adds-on
  • Added a checker for HF Radar files (NetCDF).
Corrections
  • Fix a problem on ExternalResourcesManager, that blocked the batch mode.
  • Medatlas -> CFPoint conversion: automatically update C17 labels from BODC vocabs.
  • Partially upgrade to Java 8 Time library to fix datetime handling.
  • Fix datetime management for MGD77 files.
  • Handle new L20 value.
  • Detect multiple parameters for same P09 in multistation Medatlas files (those should be splitted before conversion).
  • Fixes on labels, messages, typos, ...

Version 1.5.3 - 2020/04/08

Adds-on
  • Update URLs for CDIs, CSRs and NERC vocabularies
  • Allow DPSF P09 parameter as depth below sea surface in MedAtlas files (e.g. core data)
  • Allow exception for ODV files with empty vertical reference (e.g. navigation, some MGD files, contaminants in biota). These files must contain the following comment in the ODV header: //SeaDataNet file without vertical reference.

Version 1.5.2.0 - 2019/10/29

Corrections
  • ODV, fix bottom depth default value (set empty value instead of 0)
  • Conversion MGD to ODV: modifications on depth parameter
Adds-on
  • Use XXXX P06 unit instead of UUUU for text parameters
  • Display variant in the string "Generated By" generated by Octopus in the output file
  • Allow usage of the same P01 parameter multiple times in a file (an index is added to the NetCDF variable names if needed)

Version 1.5.1.0 - 2019/08/02

Corrections
  • ODV biological variant: allow VOLFFMXX for Sampling Effort P01 code
  • ODV format: detect an error if a mapping line does not start with // characters
  • ODV format: fix bug in variant detection (case of physico-chemical file with MINWDIST and MAXWDIST)
Adds-on
  • NetCDF SDN Format: improve validation log messages for global attributes

Version 1.5.0.6 - 2019/06/05

Corrections
  • NetCDF SDN Format: all numeric types are allowed (not only doubles)
  • Conversion MedAtlas to ODV: use of default value defined in the current station header
  • ODV format: Depth is no more mandatory in timeseries that contain a parameter belonging to P02 ASLV
  • Conversion NetCDF to ODV : remove extra tabulation at the end of lines
  • Allow COREDIST also in cm (ULCM) as primary variable
  • MedAtlas check: detect an error if a value = default and the flag != 9
Adds-on
  • Take into account ODV variants for biology, flow cytometry and microlitters for checks and conversions ODV to ODV
  • ODV check: the primary variable must have at least one non null value

Version 1.4.2 - 2019/03/04

Corrections
  • Fix error if MedAtlas file has a last empty line
  • NetCDF SDN: update global attribute 'title' : replace 'SeaDataNet_1.0 CF 1.6' with 'SeaDataNet_1.0 CF-1.6'
  • Manage errors if the user is not connected and tries to update the external resources (CSR, BODC, EDMO)
  • NetCDF SDN: fix expected variable type during check for DEPTH (any type is authorised)
  • Manage error if input directory is empty
  • Keep current window size when changing language in settings
  • Set default log level to INFO
Adds-on
  • Compliance with java 9 and 10
  • Check consistency between values and quality flags
  • Check consider as valid timeseries ODV files with a P02 ASLV parameter instead of depth
  • Homogenisation of the default value for measured parameters in NetCDF SDN (-99999)
  • Allow SDN CSR references using the central catalog
  • Non SDN MedAtlas files are recognized but considered as invalid (can be converted)
  • Conversion from MedAtlas: the SDN CSR reference is automatically added from EDMO code and cruise reference, if possible
  • All conversions: non-mandatory variables are not kept if they are empty (with only default values)
  • Conversion to ODV: an error is raised if one of the mandatory columns has missing value(s)
  • Update SDN QC flags: add 'Q' flag
  • JSON log: format added in files logs
  • Choose automatically the default output directory for conversion to mono-station files
  • Ability to update external resources (CSR, BODC, EDMO) in batch mode

Version 1.3.9 - 2017/07/21

Corrections
  • ODV files: Management of all ISO date formats for the primary variable of time series and not only [YYYY-MM-DDThh:mm:ss.sss]
  • ODV files: empty lines or lines with only tabs do not generate error.
  • ODV files: Management of UTF8 files with BOM (Byte Order Mark)
  • Allow empty SDN references using SDN_REFERENCES notation (will be ignored)
Adds-on
  • ODV files, date formats: check ISO format validity and consistency between dates an formats (metadata column headers and date primary variable in timeseries)
  • BODC terms management: detection of non existing or deprecated parameters. Automatic substitution of deprecated ones.
  • Improve log messages: display the name(s) of file(s) with error when converting a directory
  • Output file type: change 'keep/split' to 'Split to mono station files? yes/no' (no modification for batch mode)
  • Use a new Octopus version number notation: major.minor.build number
