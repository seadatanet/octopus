Journal des modifications

Version 1.4.0 - 2018/03/26

Corrections
  • Contrôleur de format MedAtlas : suppression du contrôle sur la longueur max à 80 caractères des lignes de commentaires
  • Gestion d'un "RECORD lines" faux dans les fichiers MedAtlas
  • Correction du contraste de couleur dans les menus contextuels (clic droit)
  • Génération netCDF : mise à jour de l'attribut unit si l'unité P06 est obsolète et remplacée par une autre unité
Evolutions
  • Nouvelle conversion de format : SDN netCDF vers SDN ODV
  • Nouveau contrôleur de format pour SDN netCDF
  • Amélioration du contrôleur de format SDN ODV
  • Implémentation du mode batch pour les contrôleurs de format
  • Ajout d'une information sur la création des fichiers : "Generated by Octopus [version] [date]"
  • Nom de fichier de sortie par défaut est le mÃªme que celui du fichier en entrée, l'extension est rajoutée automatiquement selon la conversion choisie
  • Utilisateur prévenu s'il se sert d'une version plus ancienne d'OCTOPUS
  • Contrôle de l'existence des références CSR dans le catalogue CSR
  • Mise à jour de la documentation avec les procédures à suivre en cas de configuration avec proxy et/ou en cas de lancement batch sans interface graphique

Version 1.3.9 - 2017/07/21

Corrections
  • Fichiers ODV, séries temporelles, date (variable primaire) : possibilité d'utiliser n'importe quel format ISO et plus seulement [YYYY-MM-DDThh:mm:ss.sss]
  • Fichiers ODV: gestion des fichiers contenant des lignes vides, ou ne contenant que des tabulations.
  • Fichiers ODV: gestion des fichiers UTF-8 avec BOM (Byte Order Mark)
  • Autorise les références SDN vides lorsqu'elles utilisent la notation SDN_REFERENCES (seront ignorées)
Evolutions
  • Fichiers ODV, formats de date: vérification de la validité des formats de dates ISO et de la cohérence entre les formats des valeurs et les formats ISO (en-têtes des colonnes date de métadonnées et date variable primaire des séries temporelles)
  • Gestion des paramètres BODC: détection des paramètres non existant ou obsolètes. Remplacement automatique des paramètres obsolètes.
  • Amélioration des journaux: affichage du(des) fichier(s) en erreur dans le cas de la conversion d'un répertoire
  • Type de fichier en sortie: remplacement de 'keep/split' par 'Découper en fichier mono-station? Oui/Non' (pas de changement pour le mode batch)
  • Utilisation d'une nouvelle notation pour le numéro de version: major.minor.build number
